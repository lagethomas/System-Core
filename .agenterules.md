# ROLE: PHP Senior Developer & System Architect (Thomas's Framework)
# CONTEXT: PHP Core System (MariaDB + Redis) - PHP 8+

## 1. ARQUITETURA E PADRÕES
- **STRICT MODE**: `declare(strict_types=1);` e tipagem obrigatória em todas as funções/métodos.
- **ESTRUTURA**: MVC Modular (Domain-Driven) com PSR-4. Versionamento semântico obrigatório.
- **PERFORMANCE**: Opcache Preloading + Redis para sessões e queries custosas (TTL 3600s).
- **AJAX-FIRST**: 100% das ações via AJAX/Fetch API. Proibido refresh de página para ações de CRUD em qualquer circunstancia e módulo do sistema, seja ele existente ou novos que estamos criando.
- **DIRETÓRIOS**:

## 2. SEGURANÇA E DADOS
- **SQL**: Uso exclusivo de PDO com Prepared Statements. SQL puro com variáveis interpoladas é proibido.
- **XSS/CSRF**: `htmlspecialchars()` em saídas HTML. Nonces (hash_hmac) obrigatórios em ações sensíveis.
- **CRIPTOGRAFIA**: Senhas com `PASSWORD_ARGON2ID`. Dados sensíveis com AES-256.
- **AMBIENTE**: Infraestrutura no `.env`. Configurações dinâmicas na tabela `settings`.

## 3. BANCO DE DADOS E MIGRATIONS
- **ATUALIZAÇÃO**: `database.sql` deve sempre refletir o estado de produção.
- **MIGRATIONS**: Alterações via `/scripts/migration.php` (idempotente).
- **EXECUÇÃO**: Exige parâmetro `?token=` validando contra `DB_MIGRATION_TOKEN`.

## 4. INTERFACE (UI/UX PREMIUM)
- **4.1. SELECTABLE CARDS (Switchers)**:
    - **Estrutura**: Container `.selectable-card`, ícone Lucide, rótulo `<span>` e `<input type="checkbox">` oculto.
    - **JS**: Gerenciado por `toggleSelectableCard(el, id, callback)` em `assets/js/components/selectable-cards.js`.
    - **Ação**: Disparar persistência AJAX imediata ao clique.
    - **Estilo**: Inativo (borda suave); Ativo (borda/detalhes `#d4af37` e fundo 10% da cor).
- **TABELAS**: Scroll horizontal obrigatório com scrollbar fina customizada.
- **MODAIS**: Estrutura [Header Fixo | Body Scroll | Footer Fixo]. Funções: `openModal(id)`/`closeModal(id)`.
- **PADRONIZAÇÃO**:
    - **Ícones**: Exclusivamente **Lucide Icons**.
    - **Botões**: Largura automática (nunca 100%).
    - **Modularização**: Proibido CSS/JS inline. Usar arquivos externos em `assets/`.

## 5. FEEDBACK E NOTIFICAÇÕES (SweetAlert2)
- **PADRÃO**: Função `notify(msg, type)` centralizada.
- **CONFIGURAÇÃO**: 
  - Posição: `top-end`.
  - Timer: 4000ms (`timerProgressBar: true`).
  - Estilo: Fundo `#1a1b26`, texto branco, detalhes em dourado `#d4af37`.

## 6. REGRAS DE NEGÓCIO E LOGS
- **AUDITORIA**: Registro obrigatório de logs CUD na tabela `cp_logs`.
- **NOTIFICAÇÕES**: Gerar registro na tabela `cp_notifications` para cada ação relevante.
- **UPLOADS**: Caminho único: `/public/uploads`.
- **SEGURANÇA**: Gestão ativa de Brute Force, retenção e bloqueio de IP.

## 7. PADRONIZAÇÃO DE FORMULÁRIOS (UI/UX PREMIUM)
- ESTRUTURA: Todo formulário deve utilizar o container .floating-group para habilitar Floating Labels.
- INPUTS: O input ou select deve vir antes da label. É obrigatório o uso de placeholder=" " (espaço vazio) para o correto funcionamento do gatilho CSS :placeholder-shown.
- GRIDS: Organização obrigatória em colunas usando .form-grid-2 (padrão) ou .form-grid-3. O sistema deve ser 100% responsivo, convertendo para 1 coluna em mobile via CSS global.
- SENHAS: Campos de senha devem estar em um .password-toggle-wrapper com botões de alternar visibilidade (UI.togglePassword) e gerador automático (UI.generatePassword).
- VALIDAÇÃO E MÁSCARAS: Uso obrigatório de classes de máscara (ex: .mask-phone, .mask-zip) e inicialização via UI.initMasks() após a renderização do formulário.
- PROCESSAMENTO: Submissão 100% via Fetch API/AJAX. Proibido o uso de action e method tradicionais que resultem em refresh de página.- BORDAS: Usar borda de 1px
