## 1. ARQUITETURA E PADRÕES
- **STRICT MODE**: `declare(strict_types=1);` e tipagem obrigatória em todas as funções/métodos.
- **ESTRUTURA**: MVC Modular (Domain-Driven) com PSR-4. Versionamento semântico obrigatório.
- **PERFORMANCE**: Opcache Preloading + Redis para sessões e queries custosas (TTL 3600s).
- **AJAX-FIRST**: 100% das ações via AJAX/Fetch API. Proibido refresh de página para ações de CRUD.
- **DIRETÓRIOS**:
  ├── src/Modules/[Dominio]/ (Controllers, Models, Views)
  ├── public/assets/css/
  │   ├── components/ (Atomic UI: buttons, forms, tables, etc.)
  │   ├── modules/ (CSS específico por módulo)
  │   ├── theme/ (Variáveis de cores/branding)
  │   └── style.css (Apenas layout/esqueleto e variáveis globais)
  └── includes/repositories/ (Abstração de dados global)

## 2. SEGURANÇA E DADOS
- **SQL**: Uso exclusivo de PDO com Prepared Statements. SQL puro com variáveis interpoladas é PROIBIDO.
- **XSS/CSRF**: `htmlspecialchars()` em saídas HTML. Tokens CSRF obrigatórios em TODOS os forms.
- **LOGS GLOBAIS**: Toda criação de log via `LogRepository` deve respeitar a configuração `security_enable_logs`.
- **CRIPTOGRAFIA**: Senhas com `PASSWORD_ARGON2ID`. Dados sensíveis com AES-256.

## 3. CSS MODULAR & COMPONENTES (PADRÃO CORE)
- **CARREGAMENTO**: O `header.php` carrega automaticamente componentes de `/assets/css/components/`. 
- **NOVOS COMPONENTES**: Devem ser registrados no loop do `header.php`.
- **PADRONIZAÇÃO OBRIGATÓRIA**:
    - **Buttons**: `.btn-primary`, `.btn-dark`, `.btn-danger`, `.btn-primary-glass`.
    - **Tables**: `.premium-table` dentro de `.table-responsive`.
    - **Switches**: Estrutura `.switch` + `.slider`.
    - **Uploads**: Padrão `.upload-selector` com `preview-img` (ver `Settings/Views/index.php`).
    - **Accordions**: Usar `accordion.css`. `.accordion-container` > `.settings-accordion` > `summary`.

## 4. UI/UX PREMIUM - REGRAS OBJETIVAS DE LAYOUT
Para garantir a consistência, toda nova página ou módulo DEVE seguir estas estruturas de HTML:

### A. ESTRUTURA DE PÁGINA (LISTAGEM/GERAL)
```html
<div class="users-header"> <!-- [modulo]-header -->
    <h2>Título da Página</h2>
    <p>Descrição curta e objetiva da funcionalidade.</p>
</div>

<div class="card">
    <div class="user-list-header"> <!-- [modulo]-list-header -->
        <h3>Subtítulo do Card</h3>
        <button class="btn-primary" onclick="openModal()">
            <i data-lucide="plus"></i> Ação Principal
        </button>
    </div>
    <div class="table-responsive">
        <table class="premium-table"> ... </table>
    </div>
</div>
```

### B. TABS COM EFEITO GLASSMORMISM (OBRIGATÓRIO)
As abas devem ficar **FORA** do card de conteúdo e usar o container `.tab-navigation`.
```html
<div class="tab-navigation">
    <a href="?tab=info" class="nav-link-tab active">
        <i data-lucide="info"></i> Informações
    </a>
    <a href="?tab=config" class="nav-link-tab">
        <i data-lucide="settings"></i> Configurações
    </a>
</div>

<div class="card card-settings"> <!-- Card de conteúdo logo abaixo das tabs -->
    <!-- Conteúdo da Tab -->
</div>
```

### C. FORMULÁRIOS COM FLOATING LABELS
Padrão inegociável para inputs e selects. O input deve ter `placeholder=" "` e a label deve vir **APÓS** o input.
```html
<div class="floating-group">
    <input type="text" name="field" class="form-control" placeholder=" ">
    <label class="floating-label">Nome do Campo</label>
</div>

<!-- Selects também usam floating labels -->
<div class="floating-group">
    <select name="status" class="form-control" placeholder=" ">
        <option value="1">Ativo</option>
    </select>
    <label class="floating-label">Status</label>
</div>
```

### D. GRIDS DE FORMULÁRIO
Use `.form-grid-2` ou `.form-grid-3` com `gap-md`.
```html
<div class="form-grid-2 gap-md">
    <div class="floating-group"> ... </div>
    <div class="floating-group"> ... </div>
</div>
```

## 5. REGRAS DE DESIGN (MODERN PREMIUM)
- **GLASSMORMISM**: Use a classe `.glassmorphism` ou backgrounds `rgba(var(--primary-rgb), 0.08)` para destaques.
- **BORDAS**: Sempre `20px` ou `24px` para cards principais. `12px` para inputs e botões.
- **SOMBRAS**: `var(--card-shadow)` para repouso, `var(--card-hover-shadow)` para hover.
- **LUCIDE ICONS**: Use sempre `<i data-lucide="icon-name"></i>`. Inicialize com `lucide.createIcons()` em modais.

## 6. PADRONIZAÇÃO DE ESCALAS (FONTES E ÍCONES)
Para manter a consistência visual "Premium", siga rigorosamente estas escalas:

### A. ÍCONES (LUCIDE)
- **PADRÃO (Tabelas/Botões)**: `16px` (Default do Core).
- **SM (Accents/Muted)**: `14px` (Use a classe `.icon-sm`).
- **BADGES (STATUS/TAGS)**: `10px` | Bold (800) | Uppercase.
    - Estrutura: `.status-badge` + `.status-[cor]`.
    - Diferenciação: Se houver >1 informação, use cores distintas (Ex: Status em Verde, Role em Azul).
- **LG (Headers/Destaques)**: `20px` ou `24px` (Para dashboards e modais).

### B. TIPOGRAFIA (FONT-SIZE)
- **CORPO/TEXTO GERAL**: `14px`.
- **SMALL (Labels/Muted/Rodapés)**: `12px` (Use a classe `.small` ou `.font-xs`).
- **TÍTULOS DE CARD (H3)**: `18px`.
- **TÍTULOS DE PÁGINA (H2)**: `24px`.
- **BADGES (STATUS/TAGS)**: `10px` (Weight 800 | Uppercase). Use a classe `.status-badge`.

### C. BADGES PREMIUM (PADRÃO DE CORES)
Para garantir a diferenciação visual, utilize as classes semânticas de status:
- **Sucesso/Ativo**: `.status-success` (Verde).
- **Atenção/Pendente**: `.status-warning` (Laranja).
- **Erro/Crítico**: `.status-danger` (Vermelho).
- **Info/Destaque**: `.status-info` (Azul).
- **Premium/Roles**: `.status-primary` (Dourado/Cor do Tema).

**Multi-informação**: Quando houver mais de uma tag (ex: Status + Nível), utilize cores diferentes para cada uma para facilitar a leitura rápida.
- Exemplo: Status em Verde (`.status-success`), Role em Azul (`.status-info`).

```html
<!-- Exemplo de Multi-Badge seguindo o padrão -->
<div class="flex gap-2">
    <span class="status-badge status-success">Ativo</span>
    <span class="status-badge status-info">Financeiro</span>
</div>
```

## 7. PADRÃO PARA CRIAÇÃO DE NOVOS MÓDULOS
Ao criar um módulo, entregue:
1. **Migration**: SQL robusto com tipos corretos.
2. **Repository**: Métodos `CRUD` + `findById` + `search`.
3. **Controller**: Retornos JSON padrão `{success: true, message: "...", data: []}`.
4. **Views**: 
    - `index.php`: Listagem com filtros e paginação.
    - Modais de Form: Uso rigoroso de `.floating-group`.
5. **No-Refresh (AJAX)**: Integração com `UI.request` e `UI.showToast`.

## 8. FEEDBACK E INTERAÇÃO
- **MENSAGENS**: Sucesso = Toast; Confirmação/Erro Crítico = SweetAlert2.
- **LOADERS**: Exibir feedback visual de carregamento em botões ao disparar AJAX.
- **ESTADO ATIVO**: Sempre marcar o item ativo na sidebar e nas abas.
- **CONSISTÊNCIA VISUAL**: Jamais use tamanhos ad-hoc (ex: `15px`, `17px`). Use sempre a escala definida no item 6.

## 9. MANUTENÇÃO (ZERO WASTE)
- Proibido CSS Inline.
- Proibido duplicar estilos de componentes em arquivos de módulo.
- Proibido manter `console.log` ou código comentado em produção.
## 10. PADRÕES DE DADOS E VALORES
- **MOEDA**: Utilizar `mask-money` em inputs e formatar via helper PHP no backend.
- **DATAS**: Inputs tipo `date` ou com máscara `.mask-date`. Exibição sempre no padrão local `d/m/Y`.
- **CONSISTÊNCIA**: Todo novo módulo deve herdar a estética "Premium" (cards, sombras, botões padronizados).
